# 错题本

把会复发的错误沉淀成未来 agent 能执行的短规则。只记录能改变后续行为的经验，不写流水账。

## 触发判断

记录满足任一条件的问题：

- 长期约定：用户明确表达“以后都要……”“以后不要……”“长期……”“记住……”“防止后面忘记”等长期偏好、流程约定或可复用规则，且这条规则会改变未来 agent 行为。
- 持久背景：用户或外部信息补充了无法或很难从当前仓库已有上下文推断的项目背景、业务状态、口径变化或约束，且未来 agent 需要知道它才能正确工作；默认自动写入项目 `AGENTS.md`。
- 高频出错：同类问题在本轮、本项目、历史记忆或用户反馈中重复出现。
- 隐蔽易犯：问题不显眼，但 agent 容易凭默认习惯再次踩坑。
- 成本较高：错误会导致返工、误操作、数据污染、错误结论、测试假阳性或线上风险。

不记录一次性环境故障、已被代码/测试/脚本硬性约束的问题、不会改变 agent 行为的普通项目事实、或已经被 AGENTS.md 清楚覆盖的内容。

## 选择写入位置

- 写项目 `AGENTS.md`：只和当前代码库、服务、测试命令、目录结构、业务约定、项目工具链或项目特有坑有关。
- 写全局 `AGENTS.md`：跨项目适用，属于用户偏好、常用平台/API、工具使用习惯、agent 工作流、通用错误模式。
- 不确定时选更窄范围；项目规则验证多次跨项目复现后再提升为全局。

定位文件：项目级优先使用当前仓库根目录的 `AGENTS.md`；没有就创建。全局级优先使用现有全局指令文件；找不到时使用 `${CODEX_HOME:-$HOME/.codex}/AGENTS.md`。

## 写入规则

先搜索目标文件，避免重复。能合并就合并，能改一句就不要新增一段。新增内容使用命令式中文，短到足够被未来 agent 快速执行。

推荐格式：

```markdown
- [范围/主题] 当 <触发条件> 时，必须/优先/不要 <具体行为>；原因：<一句话说明风险或边界>。
```

写入前问自己三件事：

- 未来 agent 看到这条，能不能少犯一次同类错误？
- 这条是不是比现有规则更具体，而不是重复提醒？
- 放在这个 AGENTS.md 的范围会不会污染无关项目？

## 完成后

简短说明写入了哪个文件、沉淀了哪类错误。不要把错题本变成复盘长文。
