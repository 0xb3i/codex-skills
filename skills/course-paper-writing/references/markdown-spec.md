# Markdown 规范

## 只维护的源文件

- `report/manuscript.md`
- `report/metadata.json`
- `report/references.bib`

## 标题

- 一级标题：`# 标题`
- 二级标题：`## 标题`
- 三级标题：`### 标题`
- 不要手写“一、二、三”或“1.1、1.2”，由底层 skill 统一生成

## 文献引用

- 使用 citekey，例如 `[@chen2024bge]`
- 不要手写 `[1]`
- 不要在正文末尾手写参考文献列表

## 图片

图片必须按下面格式书写：

```md
[[fig:system_funnel]]

![图 2 系统漏斗式架构图。](figures/02_system_funnel.png){ width=95% }
```

规则：

- 标签行必须单独成段
- 标签行后必须空一行
- 图片题注必须写在图片语法里
- 题注必须以 `图 N ...` 开头

## 表格

表格必须按下面格式书写：

```md
[[tbl:core_results]]

表 1 核心结果汇总

| 模块 / 流程 | 关键结果 | 说明 |
| --- | --- | --- |
| Embedding 初筛层 | 过滤约 60% - 70% 候选对 | 高置信相似约 10% - 20%，高置信不相似约 40% - 50% |
```

规则：

- 标签行必须单独成段
- 标签行后建议空一行
- 表题必须独立一行
- 表题必须以 `表 N ...` 开头
- 说明文字不要和表题写在同一行

## 图表交叉引用

- 引用图片：`[[ref:fig:system_funnel]]`
- 引用表格：`[[ref:tbl:core_results]]`

规则：

- `ref` 后面的标识必须和上面的标签一致
- 正文里不要手写“见图 1”“见表 2”，应优先用交叉引用占位符

## 禁止事项

- 不要手写 openxml
- 不要手写目录页
- 不要手写图目录/表目录
- 不要直接改 `report/report.docx`
