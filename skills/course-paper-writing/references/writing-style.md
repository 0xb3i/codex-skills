# 中文课程论文写作风格

## 目标

本规范用于约束课程论文正文的语言风格与内容结构。核心要求是：写出来的文本必须像学术论文，而不是像 Markdown 笔记、项目汇报提纲或产品说明文档。

## 一、语言风格

### 1. 学术化表达

- 全文必须使用正式、克制、客观的书面表达。
- 叙述应以完整段落展开，不要写成零碎条目或清单式说明。
- 优先使用“问题 - 方法 - 结果 - 解释 - 价值”的论文式叙述，而不是“点 1、点 2、点 3”的口语化展开。

### 2. 避免 Markdown 味

下列表达应尽量避免：

- 过多 bullet points
- 过多括号内插入式注释
- 过多破折号式补充说明
- 类似“如下所示：”“总结如下：”之后直接列多个短点
- 明显像笔记的压缩式句法

如果确实需要列举，优先将列举内容融入连贯段落，而不是拆成一串项目符号。

### 3. 句子与段落的流畅性

- 句子之间要有自然的逻辑衔接。
- 不要频繁使用短句堆砌。
- 不要把一段拆成很多只有一两句话的碎段。
- 每一段都应围绕一个中心意思展开，并有足够的展开长度。

### 4. 不要写成汇报口吻

避免下面这些倾向：

- 强调“我做了什么”的流水账
- 用过强的主观评价词，如“非常优秀”“极大提升”“特别有意义”
- 用产品宣发式表达包装技术结果

更合适的方式是：

- 明确任务背景
- 说明设计选择
- 给出结果
- 解释结果意味着什么

### 5. 采用 storytelling 式叙事

正文整体应呈现一种“问题驱动”的叙事，而不是把做过的事情平铺直叙地罗列出来。

推荐的叙事链条是：

- 我们面对了什么问题
- 这个问题为什么重要
- 现有做法为什么不够
- 这推动我们去设计什么方法
- 方法如何一步一步回应问题
- 结果说明这个问题是否被真正解决

也就是说，论文应当让读者感觉：

- 研究设计是被问题逼出来的
- 方法不是随意堆砌，而是服务于问题
- 实验不是机械展示，而是对问题的回答

不要写成：

- 先做了 A，再做了 B，再做了 C
- 把所有模块功能逐条掏出来
- 像项目周报一样按事项汇报

更好的写法是：

- 从问题出发
- 用问题推动方法出现
- 用结果回应问题

## 二、内容结构

### 1. 标题命名必须学术规范

标题不能随意起名，不能写成口语化、工程化或提纲式标题。

优先使用学术论文常见标题范式，例如：

- 引言
- 研究背景
- 相关工作
- 文献综述
- 任务定义
- 数据说明
- 方法设计
- 实验结果与分析
- 讨论
- 局限性与未来工作
- 结论
- 附录

不推荐的标题类型包括：

- 随意口语化标题
- 过于工程任务化的临时名字
- 为了和 Markdown 结构对齐而生造的小标题

### 2. 不要把 Markdown 层级机械搬进论文

Markdown 允许非常细的标题切分，但论文正文不应该因此被切得过碎。

必须遵守：

- 一级标题数量要克制
- 二级标题只在确有必要时出现
- 不要为了每个小点都立一个标题
- 如果一个标题下面只有半页甚至更少内容，应优先考虑并入上一层

判断标准很简单：

- 一个标题下面是否承载了一个完整的小问题
- 该部分是否值得读者在目录中单独看到
- 该部分内容是否足够支撑一个独立层级

如果答案是否定的，就不要再分标题。

### 3. 段落长度要稳定

- 段落不能太碎。
- 每个自然段应有基本展开长度。
- 一个二级标题下通常应有多个自然段，而不是只放一小段就进入下一个标题。

推荐做法：

- 每个标题先给出主题句
- 后面补充背景、细节、原因、结果或解释
- 保证该标题下内容形成一个相对完整的小节

### 4. 层次组织要服务论证

标题的作用是组织论证，不是分割 Markdown。

因此，写作时应优先考虑：

- 当前这一节要解决什么问题
- 这一节和上一节、下一节是什么关系
- 这一节是否真的需要单独成节

如果只是同一主题下的一个补充点，优先写入同一小节的段落，而不是再新增标题。

### 5. 引言、方法、结果都要有叙事功能

- 引言不是背景堆砌，而要把问题讲清楚。
- 方法不是模块清单，而要解释“为什么必须这样设计”。
- 结果不是指标陈列，而要说明“这些结果回答了什么问题”。

如果一节内容只是简单罗列做了哪些工作，而没有说明这些工作是如何被问题驱动出来的，应视为不合格写法。

### 6. 可借鉴的优秀论文叙事特征

优秀论文通常具备以下特征：

- 开篇很快把问题讲清楚，不绕远路
- 研究价值出现得早，而且是具体的
- 方法设计与问题之间有明显因果关系
- 每个章节都在推动主线，而不是独立散落
- 结果部分不仅给数字，还解释数字意味着什么
- 结论会回到最初的问题，而不是只重复方法

这类论文读起来会有明显的“推进感”，而不是“信息堆积感”。

## 三、执行要求

后续模型在撰写 `report/manuscript.md` 时，必须同时满足以下要求：

- 使用学术化、段落化语言
- 不使用过多 bullet points、括号注释、破折号补充
- 采用问题驱动、storytelling 式叙事
- 标题名称必须符合论文常见命名范式
- 不要把标题切得过碎
- 每个标题下都要有足够内容支撑
- 方法、结果、讨论都要回到问题本身

如果在写作时拿不准是否应新开一个标题，默认选择“不新开”，而是先写成同一节中的连贯段落。
